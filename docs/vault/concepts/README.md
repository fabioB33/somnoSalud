# Concepts — SomnoSalud

> Documentos de **convenciones operativas día a día** y **patrones reutilizables** del proyecto. Distinto a ADRs (que documentan **una decisión** con alternativas) — los concepts son guías vivas que pueden actualizarse sin necesitar superseding.

| Doc | Tema | Status |

## Cuándo escribir un Concept doc

- Una **regla operativa** que aplicará a >5 archivos del proyecto.
- Un **patrón reutilizable** que conviene formalizar para que múltiples desarrolladores lo apliquen igual.
- Una **convención de naming / estructura** que afecta a un package entero.

NO escribir Concept para:
- Decisiones puntuales (eso es ADR).
- Procesos cross-product (eso vive en `processes/` heredado del Pampa Labs OS).
